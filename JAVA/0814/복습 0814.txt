SVN의 단점 : 깃과 다르게 거치는 저장소가 없음

연동 설정 git init

언격저장소 가져오기 git clone git pull 등으로
그 후 편집
그담 인덱스에 일단 업 git add
로컬저장소에 커밋 git commit -m ""
---여기까진 내 PC에만 반영된 상태------------
원격저장소에 반영 git push origin master

브런치 중요! 
내가 작업할 파트를 파악해둬야됨
마스터는 절대 안만지는 걸 추천 최종결과물 구역
develop에서 보통 작업이 이루어짐
보통 브런치명은
feature - 작업명 

면접 설명: 브런치에서 내파트 파악후 디벨롭에서 피처-기능명을 정해
작업해봤다.

포크 따기 : 평행이론 개념

원본(업스트림)          포크본(오리진)

내 기존것은 보존하고 내거에 갱신본 반영시키고 포크에 다시 올림
갱신된 내용 가져올 시는 pull이 아닌 rebase를 통해 덮어 씌워줌
